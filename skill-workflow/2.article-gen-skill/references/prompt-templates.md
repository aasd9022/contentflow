# Prompt 模板库

## 1. 大纲生成 Prompt

```markdown
# 角色
你是一位资深的{style}领域内容策划专家。

# 任务
基于以下信息，生成一篇结构清晰、逻辑严密的文章大纲。

# 文章信息
- 主题：{topic}
- 目标受众：{audience}
- 写作风格：{style}
- 字数要求：{length}
- 核心要点：
  1. {key_point_1}
  2. {key_point_2}
  3. {key_point_3}
  4. {key_point_4}（如有）
  5. {key_point_5}（如有）

# 大纲要求
1. 标题：主标题 + 副标题（吸引人且准确）
2. 引言：约200字，背景引入 + 核心问题
3. 正文：3-5个主要部分，每部分包含2-3个子主题
4. 结论：约200字，核心观点 + 行动建议
5. 整体节奏符合"{style}"风格
6. 字数控制在 {length} 范围内

# 输出格式
请按以下Markdown格式输出：
```markdown
# [文章标题]

> 副标题/导读

## 引言
- 要点1
- 要点2
- 要点3

## 第一部分：[标题]
### 1.1 [子标题]
- 内容要点
### 1.2 [子标题]
- 内容要点

## 第二部分：[标题]
...

## 结论
- 核心观点
- 行动建议
```
```

## 2. 全文生成 Prompt

```markdown
# 角色
你是一位经验丰富的{style}写作者，擅长创作{audience}喜爱的内容。

# 任务
基于以下大纲，生成一篇完整的、符合要求的文章。

# 文章信息
- 主题：{topic}
- 目标受众：{audience}
- 写作风格：{style}
- 字数要求：{length}
- 核心要点：{key_points}

# 文章大纲
{outline}

# 写作要求
1. 严格遵循大纲结构，不偏离主题
2. 字数控制在 {length} 范围内（±10%）
3. 语言风格保持"{style}"，不要混用其他风格
4. 所有{key_points}都要在正文中明确体现
5. 适当使用数据、案例、引言增强说服力
6. 段落之间要有自然过渡
7. 结论部分要呼应开头的核心问题

# 输出格式
请按以下Markdown格式输出，包含YAML Front Matter：

```markdown
---
title: "[文章标题]"
date: {date}
tags: [标签1, 标签2, 标签3]
description: "[150字以内的文章摘要]"
keywords: ["关键词1", "关键词2", "关键词3"]
author: "作者"
---

# [文章标题]

> [副标题/导读]

[正文内容...]

## 引言

...

## 第一部分

...

## 结论

...
```

# 禁止事项
- 不要在文中出现"作为AI"等元描述
- 不要使用过时数据（标注数据年份）
- 不要过度使用专业术语（除非是专业风格）
- 不要超过字数上限
```

## 3. 摘要生成 Prompt

```markdown
# 任务
为以下文章生成150字以内的摘要。

# 文章内容
{article_content}

# 要求
1. 突出文章核心价值
2. 包含1-2个关键观点
3. 引发读者好奇心
4. 语言简洁有力
5. 不超过150字

# 输出格式
直接输出摘要文本，不要前缀。
```

## 4. SEO优化 Prompt

```markdown
# 任务
为以下文章生成SEO信息。

# 文章信息
- 标题：{title}
- 内容：{article_content}

# 输出要求
1. keywords：3-5个核心关键词
2. description：150字以内的SEO描述
3. tags：3-5个内容标签

# 输出格式
```yaml
keywords: [关键词1, 关键词2, 关键词3]
description: "SEO描述..."
tags: [标签1, 标签2, 标签3]
```
```

## 5. 风格转换 Prompt

```markdown
# 任务
将以下文章从「原风格」转换为「目标风格」。

# 原风格
{source_style}

# 目标风格
{target_style}

# 文章内容
{article_content}

# 要求
1. 保持原意不变
2. 调整语言风格
3. 调整段落结构（如需要）
4. 字数变化不超过±20%
5. 保留所有关键信息和数据

# 输出
转换后的完整文章。
```

## 6. 字数调整 Prompt

```markdown
# 任务
将以下文章{action}到{target_length}字。

# 原文
{article_content}

# 当前字数
{current_length}

# 调整策略
- 扩展：增加细节、例子、数据、引用
- 精简：删除冗余、合并段落、简化描述

# 要求
1. 保持原意和结构
2. 调整后字数 = {target_length}（±5%）
3. 风格保持不变
4. 关键信息不能丢失

# 输出
调整后的完整文章。
```
