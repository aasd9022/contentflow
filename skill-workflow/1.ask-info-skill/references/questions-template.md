# 信息收集问题模板

## 标准问题清单

### 必问问题（5个核心维度）

#### Q1: 主题确认
```
您想创作什么主题的文章？
请用一句话描述您要写的核心内容。
```

**追问模板**（如果用户回答模糊）：
- "您能用一个更具体的标题来表达吗？"
- "这个主题的核心讨论点是什么？"

#### Q2: 目标受众
```
您的目标读者是谁？
他们的职业背景、知识水平如何？
```

**常见受众类型**：
| 类型 | 特征 | 适用风格 |
|-----|------|---------|
| 大众读者 | 普通用户 | 通俗易懂 |
| 专业人士 | 行业从业者 | 专业严谨 |
| 学生群体 | 学习者 | 知识科普 |
| 决策者 | 管理层 | 观点评论 |
| 开发者 | 技术人群 | 技术文档 |

#### Q3: 写作风格
```
您希望文章采用什么风格？
1）专业严谨 - 学术论文、技术文档风格
2）通俗易懂 - 科普文章、大众阅读
3）故事化叙事 - 案例分析、人物故事
4）观点评论 - 时事评论、深度分析
5）教程指南 - 步骤说明、操作指导
```

#### Q4: 文章长度
```
您期望的文章字数范围？
1）短文：500-1000字（适合快讯、简评）
2）中等：1500-2000字（适合专题文章）
3）长文：3000-5000字（适合深度分析）
4）超长：5000字以上（适合白皮书）
```

#### Q5: 核心要点
```
您希望文章涵盖哪些核心要点？
请列出3-5个您认为必须讨论的内容。
```

**示例格式**：
```
要点1：技术原理与实现机制
要点2：行业应用场景分析
要点3：未来发展趋势预测
要点4：面临的挑战与机遇
```

## 可选追问问题

### 进阶信息（视情况询问）

#### Q6: 特殊要求
```
文章是否需要包含：
- 具体的数据或案例？
- 引用某位专家的观点？
- 配图或图表说明？
- 特定的语言风格（如幽默、严肃）？
```

#### Q7: 发布渠道
```
文章将发布在哪里？
- 个人博客
- 微信公众号
- 知乎/简书
- 行业媒体
- 公司官网
```

#### Q8: SEO关键词（可选）
```
是否有需要重点优化的关键词？
```

## 信息确认模板

收集完成后，使用以下模板向用户确认：

```
【信息确认】
━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 主题：{topic}
👥 受众：{audience}
✍️ 风格：{style}
📏 长度：{length}
🎯 要点：
   1. {key_point_1}
   2. {key_point_2}
   3. {key_point_3}
━━━━━━━━━━━━━━━━━━━━━━━━━━
请确认以上信息是否准确？
如需修改请直接告知。
```

## 异常情况处理

### 情况1: 用户回答模糊
- **应对策略**：使用具体例子引导
- **示例**："您是说像《xxx》这样的文章风格吗？"

### 情况2: 用户不知道想要什么
- **应对策略**：提供3-4个常见方案让用户选择
- **示例**："我推荐以下方案...您更倾向哪一种？"

### 情况3: 用户中途修改
- **应对策略**：立即更新已保存的JSON文件
- **保持灵活性**：不强制按顺序，可以跳维度修改

### 情况4: 信息收集完成
- **应对策略**：向用户复述一遍 + 输出JSON + 进入下一Skill
