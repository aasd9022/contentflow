# Ask Info Skill - 信息收集

## 功能描述

本Skill作为文章生成工作流的**第一阶段**，负责与用户进行结构化对话，收集完整的文章创作需求信息，并将结果以JSON格式输出，供下一阶段（文章生成）使用。

## 触发条件

当用户表达以下意图时自动触发：
- "我想写一篇文章..."
- "帮我写一篇关于...的文章"
- "我需要创作内容..."
- "开始新的内容项目..."

## 使用方法

### 1. 启动对话收集

通过5个核心维度收集信息：

| 维度 | 说明 | 示例 |
|-----|------|------|
| **主题（topic）** | 文章核心主题 | "AI在医疗健康领域的应用" |
| **受众（audience）** | 目标读者群体 | "医疗从业者、产品经理" |
| **风格（style）** | 文章写作风格 | "专业严谨、通俗易懂" |
| **长度（length）** | 期望文章字数 | "1500-2000字" |
| **核心要点（key_points）** | 必须涵盖的内容 | ["技术原理", "应用场景", "未来趋势"] |

### 2. 引导式提问模板

按照以下顺序与用户交互：

```
第1轮：确认主题
"您想创作什么主题的文章？请用一句话描述。"

第2轮：明确受众
"您的目标读者是谁？他们的背景如何？"

第3轮：选择风格
"您希望文章是什么风格？
 - 1）专业严谨（学术/技术文档）
 - 2）通俗易懂（科普/大众阅读）
 - 3）故事化叙事（案例/人物）
 - 4）观点评论（时评/分析）"

第4轮：确认长度
"您期望的文章字数范围？
 - 短文：500-1000字
 - 中等：1500-2000字
 - 长文：3000-5000字"

第5轮：核心要点
"您希望文章涵盖哪些核心要点？（列出3-5个）"
```

### 3. 信息确认与输出

将收集的信息整理为JSON格式并保存：

```json
{
  "topic": "文章主题",
  "audience": "目标受众",
  "style": "写作风格",
  "length": "字数范围",
  "key_points": ["要点1", "要点2", "要点3"],
  "timestamp": "2026-06-25T19:45:00",
  "session_id": "唯一标识"
}
```

## 调用方式

```bash
# 通过Skill工具调用
Skill: ask-info-skill

# 或在对话中提及
"使用 ask-info-skill 收集信息"
```

## 输出文件

信息收集完成后，会在工作流目录生成：
```
skill-workflow/
└── data/
    └── article-info-{timestamp}.json
```

## 与其他Skill的协作

- **下游Skill**：`2.article-gen-skill`（文章生成）
- **数据传递**：将JSON文件路径作为参数传递给下一阶段
- **协作模式**：流水线式串行处理

## 最佳实践

1. **分轮提问**：不要一次抛出所有问题，每轮只问1-2个
2. **提供选项**：对风格、长度等维度提供预定义选项
3. **确认理解**：收集完成后向用户复述一遍，确认无误
4. **允许修改**：允许用户随时修改已收集的信息
5. **保存进度**：每完成一个维度立即保存，避免信息丢失
